# List Rendering

`v-for` renders a list of elements based on an array, object, or number range.

## Basic v-for

### Arrays

```vue
<script setup>
const fruits = ref(['Apple', 'Banana', 'Cherry'])
</script>

<template>
  <ul>
    <li v-for="(fruit, index) in fruits" :key="fruit">
      {{ index }}: {{ fruit }}
    </li>
  </ul>
</template>
```

### Objects

```vue
<script setup>
const user = reactive({ name: 'Alice', age: 25, role: 'Developer' })
</script>

<template>
  <div v-for="(value, key, index) in user" :key="key">
    {{ index }}. {{ key }}: {{ value }}
  </div>
</template>
```

### Number Ranges

```vue
<template>
  <!-- Renders 1 through 10 -->
  <span v-for="n in 10" :key="n">{{ n }} </span>
</template>
```

## The Key Attribute

Always provide a unique `key` with `v-for`. It helps Vue track element identity for efficient DOM updates:

```vue
<!-- GOOD — unique stable key -->
<li v-for="item in items" :key="item.id">{{ item.name }}</li>

<!-- BAD — index changes when items reorder -->
<li v-for="(item, index) in items" :key="index">{{ item.name }}</li>
```

Without `key` (or using index), Vue patches elements in-place. This causes bugs when list items have local state (inputs, animations).

## Array Mutation Methods

Vue detects calls to these mutation methods and triggers updates:

| Method | What it does |
|---|---|
| `push()` | Adds to end |
| `pop()` | Removes from end |
| `shift()` | Removes from start |
| `unshift()` | Adds to start |
| `splice()` | Add/remove at position |
| `sort()` | Sorts in place |
| `reverse()` | Reverses in place |

```vue
<script setup>
const items = ref([{ id: 1, name: 'A' }, { id: 2, name: 'B' }])

function addItem() {
  items.value.push({ id: Date.now(), name: 'New' })
}

function removeFirst() {
  items.value.shift()
}
</script>
```

### Replacing an Array

Non-mutating methods (`filter`, `map`, `slice`) return new arrays. Assign the result back:

```vue
<script setup>
const numbers = ref([1, 2, 3, 4, 5])

function removeEvens() {
  numbers.value = numbers.value.filter(n => n % 2 !== 0)
}
</script>
```

Vue is smart about DOM reuse — it won't re-render elements that haven't changed.

## Computed Filtered/Sorted Lists

Use computed properties instead of modifying the source array:

```vue
<script setup>
const items = ref([
  { name: 'Banana', price: 2 },
  { name: 'Apple', price: 3 },
  { name: 'Cherry', price: 5 },
])
const search = ref('')

const filteredItems = computed(() =>
  items.value
    .filter(item => item.name.toLowerCase().includes(search.value.toLowerCase()))
    .sort((a, b) => a.price - b.price)
)
</script>

<template>
  <input v-model="search" placeholder="Search...">
  <ul>
    <li v-for="item in filteredItems" :key="item.name">
      {{ item.name }} — ${{ item.price }}
    </li>
  </ul>
</template>
```

## v-for on `<template>`

Render multiple elements per iteration without a wrapper:

```vue
<template>
  <template v-for="item in items" :key="item.id">
    <dt>{{ item.term }}</dt>
    <dd>{{ item.definition }}</dd>
  </template>
</template>
```

## v-for with Components

Pass item data via props:

```vue
<template>
  <TodoItem
    v-for="todo in todos"
    :key="todo.id"
    :title="todo.title"
    :done="todo.done"
    @toggle="todo.done = !todo.done"
  />
</template>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Missing `:key` | Always provide a unique, stable key |
| Using index as key with reorderable lists | Use item ID or unique property |
| `v-if` on same element as `v-for` | Use `<template v-for>` with nested `v-if` |
| Directly mutating a prop array | Emit an event to the parent instead |
| Modifying array during iteration | Use computed to filter, not `v-if` inline |
