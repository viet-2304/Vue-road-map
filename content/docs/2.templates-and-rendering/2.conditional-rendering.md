# Conditional Rendering

Vue provides two main ways to conditionally render elements: `v-if` and `v-show`.

## v-if / v-else / v-else-if

Conditionally render elements by adding or removing them from the DOM:

```vue
<script setup>
const score = ref(85)
</script>

<template>
  <p v-if="score >= 90">Excellent!</p>
  <p v-else-if="score >= 70">Good job</p>
  <p v-else-if="score >= 50">Keep trying</p>
  <p v-else>Needs improvement</p>
</template>
```

`v-else` and `v-else-if` must immediately follow a `v-if` or `v-else-if` element.

## v-if on `<template>`

To toggle multiple elements without adding a wrapper, use `v-if` on `<template>`:

```vue
<template>
  <template v-if="isLoggedIn">
    <h1>Welcome back!</h1>
    <p>Your dashboard</p>
    <nav>...</nav>
  </template>
  <template v-else>
    <h1>Please sign in</h1>
    <LoginForm />
  </template>
</template>
```

The `<template>` element is not rendered in the DOM.

## v-show

Toggles the element's `display` CSS property. The element is always in the DOM:

```vue
<template>
  <p v-show="isVisible">I'm always in the DOM, just hidden</p>
</template>
```

## v-if vs v-show

| Feature | `v-if` | `v-show` |
|---|---|---|
| DOM behavior | Adds/removes element | Toggles `display: none` |
| Initial cost | Lazy — skips if false | Always renders |
| Toggle cost | Higher (destroy/recreate) | Lower (CSS only) |
| Supports `<template>` | Yes | No |
| Supports `v-else` | Yes | No |

**Rule of thumb:**
- **`v-show`** for elements that toggle frequently (tabs, toggles)
- **`v-if`** for conditions that rarely change, or for expensive components

## Key for Element Reuse

Vue reuses elements for efficiency. Use `key` to force recreation:

```vue
<script setup>
const loginType = ref('username')
</script>

<template>
  <template v-if="loginType === 'username'">
    <label>Username</label>
    <input key="username" placeholder="Enter username">
  </template>
  <template v-else>
    <label>Email</label>
    <input key="email" placeholder="Enter email">
  </template>
  <button @click="loginType = loginType === 'username' ? 'email' : 'username'">
    Toggle login type
  </button>
</template>
```

Without `key`, Vue reuses the `<input>` and user input carries over between toggles.

## v-if with v-for

Never use `v-if` and `v-for` on the same element. `v-if` has higher priority and won't have access to the loop variable:

```vue
<!-- BAD — v-if can't access 'item' -->
<li v-for="item in list" v-if="item.active">{{ item.name }}</li>

<!-- GOOD — use computed or wrap with <template> -->
<template v-for="item in list" :key="item.id">
  <li v-if="item.active">{{ item.name }}</li>
</template>

<!-- BETTER — use computed to filter -->
<li v-for="item in activeItems" :key="item.id">{{ item.name }}</li>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| `v-else` not adjacent to `v-if` | Must be the immediate sibling |
| `v-if` + `v-for` on same element | Use `<template>` wrapper or computed |
| Using `v-show` for expensive components | Use `v-if` to avoid initial render cost |
| Input state persisting across `v-if` toggle | Add unique `key` attributes |
