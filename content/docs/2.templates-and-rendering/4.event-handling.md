# Event Handling

Vue uses the `v-on` directive (shorthand `@`) to listen to DOM events and run JavaScript.

## Inline vs Method Handlers

### Inline Handlers

Simple expressions directly in the template:

```vue
<script setup>
const count = ref(0)
</script>

<template>
  <button @click="count++">Count: {{ count }}</button>
</template>
```

### Method Handlers

Reference a function for complex logic:

```vue
<script setup>
const name = ref('')

function greet() {
  alert(`Hello, ${name.value}!`)
}
</script>

<template>
  <input v-model="name">
  <button @click="greet">Greet</button>
</template>
```

### Calling Methods with Arguments

```vue
<script setup>
function say(message: string) {
  alert(message)
}
</script>

<template>
  <button @click="say('Hello')">Say Hello</button>
  <button @click="say('Bye')">Say Bye</button>
</template>
```

## Accessing the Event Object

Method handlers receive the native DOM event automatically:

```vue
<script setup>
function handleClick(event: MouseEvent) {
  console.log(event.target)
  console.log(event.clientX, event.clientY)
}
</script>

<template>
  <button @click="handleClick">Click me</button>
</template>
```

When calling with arguments, pass `$event` explicitly:

```vue
<template>
  <button @click="handleClick('hello', $event)">Click</button>
</template>
```

## Event Modifiers

Modifiers handle common event patterns declaratively:

```vue
<template>
  <!-- Prevent default behavior -->
  <form @submit.prevent="onSubmit">...</form>

  <!-- Stop event propagation -->
  <button @click.stop="handleClick">Click</button>

  <!-- Only trigger once -->
  <button @click.once="initialize">Init</button>

  <!-- Only if target is the element itself -->
  <div @click.self="handleSelf">
    <button>Clicking here won't trigger handleSelf</button>
  </div>

  <!-- Chaining modifiers -->
  <a @click.stop.prevent="handler">Link</a>

  <!-- Passive — improves scroll performance -->
  <div @scroll.passive="onScroll">...</div>
</template>
```

### Modifier Reference

| Modifier | Effect |
|---|---|
| `.stop` | `event.stopPropagation()` |
| `.prevent` | `event.preventDefault()` |
| `.self` | Only trigger if `event.target === element` |
| `.once` | Trigger at most once |
| `.passive` | Marks as passive listener |
| `.capture` | Use capture mode |

## Key Modifiers

Listen for specific keyboard keys:

```vue
<template>
  <!-- Only on Enter key -->
  <input @keyup.enter="submit">

  <!-- Only on Escape key -->
  <input @keyup.esc="cancel">

  <!-- Ctrl + Enter -->
  <textarea @keyup.ctrl.enter="submit"></textarea>

  <!-- Any key alias -->
  <input @keyup.tab="nextField">
  <input @keyup.delete="clear">
  <input @keyup.space="toggle">
  <input @keyup.up="increment">
  <input @keyup.down="decrement">
</template>
```

### System Modifier Keys

```vue
<template>
  <!-- Ctrl + Click -->
  <div @click.ctrl="selectMultiple">Select</div>

  <!-- Alt + Enter -->
  <input @keyup.alt.enter="altSubmit">

  <!-- Exact — only Ctrl, no other keys -->
  <button @click.ctrl.exact="onCtrlClick">Ctrl+Click only</button>

  <!-- No system modifiers at all -->
  <button @click.exact="onClick">Plain click only</button>
</template>
```

## Mouse Button Modifiers

```vue
<template>
  <div @click.left="onLeftClick">Left click</div>
  <div @click.right="onRightClick">Right click</div>
  <div @click.middle="onMiddleClick">Middle click</div>
</template>
```

## Custom Component Events

Components emit custom events:

```vue
<!-- ChildComponent.vue -->
<script setup>
const emit = defineEmits(['update', 'delete'])
</script>

<template>
  <button @click="emit('update', newValue)">Update</button>
  <button @click="emit('delete')">Delete</button>
</template>

<!-- ParentComponent.vue -->
<template>
  <ChildComponent @update="handleUpdate" @delete="handleDelete" />
</template>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Calling function instead of referencing | `@click="handler"` not `@click="handler()"` (unless passing args) |
| `.prevent` on non-default events | Only meaningful for events with default behavior (submit, click on links) |
| `.passive` with `.prevent` | Cannot combine — passive tells browser you won't prevent |
| Forgetting `$event` with inline args | Pass `$event` explicitly when needed |
