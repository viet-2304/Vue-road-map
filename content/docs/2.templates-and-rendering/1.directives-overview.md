# Directives Overview

Directives are special attributes prefixed with `v-` that apply reactive behavior to the DOM.

## Directive Syntax

```
v-directive:argument.modifier="expression"
```

Example: `v-on:click.prevent="handleSubmit"` — directive `v-on`, argument `click`, modifier `.prevent`, expression `handleSubmit`.

## v-text

Sets the element's text content, replacing any existing content:

```vue
<script setup>
const message = ref('Hello World')
</script>

<template>
  <!-- These are equivalent -->
  <span v-text="message"></span>
  <span>{{ message }}</span>

  <!-- But v-text replaces ALL content -->
  <p v-text="name"></p>

  <!-- Interpolation can mix with static text -->
  <p>Hello, {{ name }}!</p>
</template>
```

## v-html

Renders raw HTML inside an element:

```vue
<script setup>
const richContent = ref('<strong>Bold</strong> and <em>italic</em>')
</script>

<template>
  <div v-html="richContent"></div>
</template>
```

**Security Warning:** Never use `v-html` with user-provided content — it opens the door to XSS attacks:

```vue
<!-- DANGEROUS -->
<div v-html="userComment"></div>

<!-- SAFE — always use text interpolation for user input -->
<div>{{ userComment }}</div>
```

## v-cloak

Hides un-compiled template syntax until Vue takes over. Pair with CSS to prevent flash of `{{ }}`:

```vue
<template>
  <div v-cloak>{{ message }}</div>
</template>

<style>
[v-cloak] { display: none; }
</style>
```

Rarely needed with modern build tools since templates are pre-compiled.

## v-pre

Skips compilation — displays raw mustache syntax:

```vue
<template>
  <span v-pre>{{ this will not be compiled }}</span>
</template>
```

Useful when showing Vue template syntax in documentation.

## v-once

Renders once and never updates, even when reactive data changes:

```vue
<script setup>
const count = ref(0)
</script>

<template>
  <h1 v-once>Initial: {{ count }}</h1>  <!-- Never updates -->
  <p>Current: {{ count }}</p>            <!-- Updates normally -->
  <button @click="count++">+1</button>
</template>
```

## v-memo

Memoizes a template sub-tree. Only re-renders when a listed dependency changes:

```vue
<template>
  <div v-memo="[valueA, valueB]">
    <ExpensiveComponent :a="valueA" :b="valueB" :c="valueC" />
  </div>
</template>
```

Particularly useful inside `v-for` with large lists:

```vue
<template>
  <div
    v-for="item in list"
    :key="item.id"
    v-memo="[item.name, selected === item.id]"
  >
    <p>{{ item.name }}</p>
    <span>{{ selected === item.id ? 'Active' : '' }}</span>
  </div>
</template>
```

`v-memo="[]"` is equivalent to `v-once`.

## Shorthand Reference

| Directive | Shorthand | Example |
|---|---|---|
| `v-bind` | `:` | `:href="url"` |
| `v-on` | `@` | `@click="handler"` |
| `v-slot` | `#` | `#header` |

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| `v-html` with user input → XSS | Always sanitize or use text interpolation |
| `v-once` on content you expect to update | Only use for truly static initial values |
| `v-memo` with missing dependencies | List ALL reactive values used in the block |
