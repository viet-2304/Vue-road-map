# Lifecycle Hooks

Every Vue component goes through a series of steps: creation, mounting, updating, and destruction. Lifecycle hooks let you run code at each stage.

## Lifecycle Diagram

```
setup()
  ↓
onBeforeMount  →  onMounted
  ↓                  ↓
onBeforeUpdate →  onUpdated    (when reactive state changes)
  ↓
onBeforeUnmount → onUnmounted
```

## Core Hooks

### onMounted

Runs after the component is inserted into the DOM. Use for DOM access, API calls, and third-party library initialization:

```vue
<script setup>
const canvas = ref<HTMLCanvasElement>()

onMounted(() => {
  // DOM is available here
  const ctx = canvas.value?.getContext('2d')
  ctx?.fillRect(0, 0, 100, 100)
})
</script>

<template>
  <canvas ref="canvas"></canvas>
</template>
```

### onUpdated

Runs after reactive state changes cause a DOM update:

```vue
<script setup>
const messages = ref<string[]>([])
const container = ref<HTMLDivElement>()

onUpdated(() => {
  // Auto-scroll to bottom when messages change
  if (container.value) {
    container.value.scrollTop = container.value.scrollHeight
  }
})
</script>
```

**Warning:** Don't modify reactive state inside `onUpdated` — it can cause infinite loops.

### onUnmounted

Runs when the component is removed from the DOM. Use for cleanup:

```vue
<script setup>
let intervalId: number

onMounted(() => {
  intervalId = window.setInterval(() => {
    console.log('tick')
  }, 1000)
})

onUnmounted(() => {
  clearInterval(intervalId)  // Prevent memory leaks
})
</script>
```

### onBeforeMount / onBeforeUpdate / onBeforeUnmount

Run just before the corresponding stage:

```vue
<script setup>
onBeforeMount(() => {
  console.log('About to mount — DOM not ready yet')
})

onBeforeUpdate(() => {
  console.log('About to re-render — can read current DOM state')
})

onBeforeUnmount(() => {
  console.log('About to unmount — component still functional')
})
</script>
```

## onErrorCaptured

Catches errors from descendant components:

```vue
<script setup>
const error = ref<Error | null>(null)

onErrorCaptured((err, instance, info) => {
  error.value = err as Error
  console.error(`Error in ${info}:`, err)
  return false  // Prevent error from propagating further
})
</script>

<template>
  <div v-if="error" class="error">Something went wrong: {{ error.message }}</div>
  <slot v-else />
</template>
```

## Execution Order

### Parent vs Child

```
Parent setup()
Parent onBeforeMount
  Child setup()
  Child onBeforeMount
  Child onMounted
Parent onMounted
```

Unmounting is reversed:

```
Parent onBeforeUnmount
  Child onBeforeUnmount
  Child onUnmounted
Parent onUnmounted
```

### Multiple Hooks of the Same Type

They run in registration order:

```vue
<script setup>
onMounted(() => console.log('first'))
onMounted(() => console.log('second'))
// Output: "first", then "second"
</script>
```

## Cleanup Patterns

### Event Listeners

```vue
<script setup>
function handleResize() {
  console.log(window.innerWidth)
}

onMounted(() => window.addEventListener('resize', handleResize))
onUnmounted(() => window.removeEventListener('resize', handleResize))
</script>
```

### WebSocket Connection

```vue
<script setup>
let ws: WebSocket

onMounted(() => {
  ws = new WebSocket('wss://api.example.com')
  ws.onmessage = (event) => {
    messages.value.push(JSON.parse(event.data))
  }
})

onUnmounted(() => {
  ws?.close()
})
</script>
```

### Using Composables for Cleanup

Extract cleanup logic into composables:

```ts
// composables/useInterval.ts
export function useInterval(callback: () => void, ms: number) {
  const id = ref<number>()

  onMounted(() => {
    id.value = window.setInterval(callback, ms)
  })

  onUnmounted(() => {
    if (id.value) clearInterval(id.value)
  })
}
```

```vue
<script setup>
const count = ref(0)
useInterval(() => count.value++, 1000)
</script>
```

## SSR Considerations

In SSR/SSG environments (like Nuxt), some hooks behave differently:

| Hook | Runs on Server? | Notes |
|---|---|---|
| `setup()` | Yes | Runs on both server and client |
| `onBeforeMount` | No | Client only |
| `onMounted` | No | Client only — safe for DOM/browser APIs |
| `onServerPrefetch` | Yes | Server only — for async data fetching |

```vue
<script setup>
// This runs on both server and client
const data = ref(null)

// This only runs in the browser
onMounted(() => {
  window.addEventListener('scroll', handleScroll)
})
</script>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Accessing DOM in `setup()` | Wait for `onMounted` |
| Forgetting cleanup in `onUnmounted` | Always clean up timers, listeners, connections |
| Modifying state in `onUpdated` | Causes infinite update loops — use `watch` instead |
| Using browser APIs during SSR | Wrap in `onMounted` or check `import.meta.client` |
