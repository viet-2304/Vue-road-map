# Computed Properties

Computed properties derive values from reactive state. They cache results and only recalculate when dependencies change.

## Basic computed()

```vue
<script setup>
const firstName = ref('John')
const lastName = ref('Doe')

const fullName = computed(() => `${firstName.value} ${lastName.value}`)
</script>

<template>
  <p>{{ fullName }}</p>
</template>
```

`fullName` updates automatically when `firstName` or `lastName` changes.

## Computed vs Methods

```vue
<script setup>
const items = ref([1, 2, 3, 4, 5])

// Computed — cached, recalculates only when items changes
const evenCount = computed(() => items.value.filter(n => n % 2 === 0).length)

// Method — runs every time it's called
function getEvenCount() {
  return items.value.filter(n => n % 2 === 0).length
}
</script>

<template>
  <!-- These render the same, but computed is cached -->
  <p>{{ evenCount }}</p>
  <p>{{ evenCount }}</p>  <!-- Uses cache, no recalculation -->

  <p>{{ getEvenCount() }}</p>
  <p>{{ getEvenCount() }}</p>  <!-- Runs the function again -->
</template>
```

**Rule:** Use computed for derived state. Use methods for actions or when you need parameters.

## Writable Computed

Provide both `get` and `set`:

```vue
<script setup>
const firstName = ref('John')
const lastName = ref('Doe')

const fullName = computed({
  get: () => `${firstName.value} ${lastName.value}`,
  set: (newValue: string) => {
    const [first, ...rest] = newValue.split(' ')
    firstName.value = first
    lastName.value = rest.join(' ')
  }
})
</script>

<template>
  <input v-model="fullName">
  <p>First: {{ firstName }}, Last: {{ lastName }}</p>
</template>
```

## Computed with Arrays

Common patterns for filtering and sorting:

```vue
<script setup>
const todos = ref([
  { id: 1, text: 'Learn Vue', done: true },
  { id: 2, text: 'Build app', done: false },
  { id: 3, text: 'Deploy', done: false },
])

const pendingTodos = computed(() => todos.value.filter(t => !t.done))
const completedCount = computed(() => todos.value.filter(t => t.done).length)
const progress = computed(() =>
  Math.round((completedCount.value / todos.value.length) * 100)
)
</script>

<template>
  <p>Progress: {{ progress }}%</p>
  <ul>
    <li v-for="todo in pendingTodos" :key="todo.id">{{ todo.text }}</li>
  </ul>
</template>
```

## Chaining Computed Properties

Computed properties can depend on other computed properties:

```vue
<script setup>
const items = ref([{ price: 10 }, { price: 20 }, { price: 30 }])
const taxRate = ref(0.1)

const subtotal = computed(() => items.value.reduce((sum, i) => sum + i.price, 0))
const tax = computed(() => subtotal.value * taxRate.value)
const total = computed(() => subtotal.value + tax.value)
</script>
```

## Best Practices

1. **Keep computed getters side-effect free** — no async calls, no DOM mutations, no other state changes
2. **Don't mutate computed values** — the returned value is a derived snapshot (unless writable)
3. **Prefer computed over watchers** for derived state
4. **Avoid expensive operations** without caching — that's what computed is for

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Async operations inside computed | Use `watch` or `watchEffect` instead |
| Modifying reactive state in getter | Keep getters pure — no side effects |
| Returning same reference (array/object) | Computed caches by reference; return new arrays from filter/map |
| Not using `.value` in `<script>` | `computed()` returns a ref — access via `.value` in JS |
