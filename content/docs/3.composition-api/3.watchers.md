# Watchers

Watchers let you run side effects when reactive state changes — API calls, DOM manipulation, logging, etc.

## watch()

Watch specific reactive sources and run a callback when they change:

```vue
<script setup>
const query = ref('')
const results = ref([])

watch(query, async (newQuery, oldQuery) => {
  console.log(`Changed from "${oldQuery}" to "${newQuery}"`)
  if (newQuery.length > 2) {
    results.value = await searchAPI(newQuery)
  }
})
</script>
```

### Watch Sources

```ts
// Single ref
watch(count, (newVal) => { ... })

// Getter function
watch(() => user.name, (newName) => { ... })

// Multiple sources
watch([firstName, lastName], ([newFirst, newLast], [oldFirst, oldLast]) => {
  console.log(`Name changed to ${newFirst} ${newLast}`)
})

// Reactive object property
watch(() => state.user.address, (newAddr) => { ... })
```

## Options

### immediate

Run the callback immediately with the current value:

```ts
watch(source, (val) => {
  fetchData(val)
}, { immediate: true })
```

### deep

Watch nested properties of a reactive object:

```ts
const user = reactive({ profile: { name: 'Alice', settings: { theme: 'dark' } } })

// Without deep, won't trigger on nested changes
watch(() => user.profile, (newProfile) => {
  console.log('Profile changed')
}, { deep: true })
```

**Note:** Watching a `reactive()` object directly is automatically deep. Watching a getter `() => obj` is not.

### once (Vue 3.4+)

Trigger only once, then auto-stop:

```ts
watch(source, (val) => {
  initializeWith(val)
}, { once: true })
```

## watchEffect()

Automatically tracks all reactive dependencies used inside the callback:

```vue
<script setup>
const userId = ref(1)
const userData = ref(null)

watchEffect(async () => {
  const response = await fetch(`/api/users/${userId.value}`)
  userData.value = await response.json()
})
</script>
```

No need to specify sources — Vue detects them automatically. Runs immediately on creation.

### watch vs watchEffect

| Feature | `watch` | `watchEffect` |
|---|---|---|
| Explicit sources | Yes | No (auto-tracked) |
| Access old value | Yes | No |
| Lazy by default | Yes | No (runs immediately) |
| Best for | Specific reactions | Side effects with multiple deps |

## Cleanup

Handle cleanup for async operations or subscriptions:

```ts
watch(id, async (newId, oldId, onCleanup) => {
  const controller = new AbortController()

  onCleanup(() => {
    controller.abort()  // Cancel previous request
  })

  const data = await fetch(`/api/${newId}`, { signal: controller.signal })
  result.value = await data.json()
})

// watchEffect version
watchEffect((onCleanup) => {
  const controller = new AbortController()
  onCleanup(() => controller.abort())

  fetch(`/api/${id.value}`, { signal: controller.signal })
    .then(r => r.json())
    .then(data => result.value = data)
})
```

## Timing: Pre, Post, Sync

```ts
// Default — runs before DOM update
watch(source, callback)

// After DOM update — use when you need updated DOM
watch(source, callback, { flush: 'post' })
// Or use the shorthand:
watchPostEffect(() => {
  // DOM is updated here
})

// Synchronous — use rarely, runs immediately on change
watch(source, callback, { flush: 'sync' })
```

## Stopping a Watcher

Watchers created in `setup()` are auto-stopped on component unmount. For manual control:

```ts
const stop = watch(source, callback)

// Later...
stop()  // Stop watching
```

```ts
const stop = watchEffect(() => { ... })
stop()
```

## Practical Example: Debounced Search

```vue
<script setup>
const query = ref('')
const results = ref([])
const loading = ref(false)

watch(query, (newQuery) => {
  loading.value = true
  debouncedSearch(newQuery)
})

const debouncedSearch = useDebounceFn(async (q: string) => {
  if (!q.trim()) {
    results.value = []
    loading.value = false
    return
  }
  results.value = await searchAPI(q)
  loading.value = false
}, 300)
</script>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Watching a reactive object getter without `deep` | Add `{ deep: true }` or watch specific property |
| Infinite loops from modifying watched source | Use a flag or watch a different source |
| Async operations not cancelled on re-trigger | Use `onCleanup` with AbortController |
| Using `watchEffect` when you need old value | Switch to `watch` |
| Not stopping manual watchers | Stop watchers created outside setup or conditionally |
