# Pinia State Management

Pinia is the official state management library for Vue. It replaces Vuex with a simpler, TypeScript-friendly API.

## Defining a Store

### Setup Syntax (Recommended)

```ts
// stores/counter.ts
export const useCounterStore = defineStore('counter', () => {
  // State
  const count = ref(0)
  const name = ref('Counter')

  // Getters (computed)
  const doubleCount = computed(() => count.value * 2)

  // Actions (functions)
  function increment() {
    count.value++
  }

  async function fetchCount() {
    const res = await fetch('/api/count')
    count.value = await res.json()
  }

  return { count, name, doubleCount, increment, fetchCount }
})
```

### Options Syntax

```ts
export const useCounterStore = defineStore('counter', {
  state: () => ({ count: 0, name: 'Counter' }),
  getters: {
    doubleCount: (state) => state.count * 2,
  },
  actions: {
    increment() { this.count++ },
  },
})
```

## Using a Store

```vue
<script setup>
const counter = useCounterStore()
</script>

<template>
  <p>Count: {{ counter.count }}</p>
  <p>Double: {{ counter.doubleCount }}</p>
  <button @click="counter.increment()">+1</button>
</template>
```

### Destructuring with storeToRefs

```vue
<script setup>
const counter = useCounterStore()

// WRONG — loses reactivity
const { count } = counter

// CORRECT — preserves reactivity for state/getters
const { count, doubleCount } = storeToRefs(counter)

// Actions can be destructured directly
const { increment } = counter
</script>
```

## Store-to-Store Interaction

```ts
export const useCartStore = defineStore('cart', () => {
  const items = ref([])
  const userStore = useUserStore()  // Use another store

  const total = computed(() => {
    const subtotal = items.value.reduce((sum, i) => sum + i.price, 0)
    return userStore.isPremium ? subtotal * 0.9 : subtotal
  })

  return { items, total }
})
```

## $subscribe — Watch State Changes

```ts
const counter = useCounterStore()

counter.$subscribe((mutation, state) => {
  console.log('Type:', mutation.type)   // 'direct' | 'patch object' | 'patch function'
  console.log('New state:', state)

  // Persist to localStorage
  localStorage.setItem('counter', JSON.stringify(state))
})
```

## $onAction — Watch Actions

```ts
counter.$onAction(({ name, args, after, onError }) => {
  console.log(`Action "${name}" called with`, args)

  after((result) => {
    console.log(`Action "${name}" finished with`, result)
  })

  onError((error) => {
    console.error(`Action "${name}" failed:`, error)
  })
})
```

## Resetting State

```ts
const counter = useCounterStore()
counter.$reset()  // Resets to initial state (Options syntax only)

// For Setup syntax, define your own reset:
export const useCounterStore = defineStore('counter', () => {
  const count = ref(0)

  function $reset() {
    count.value = 0
  }

  return { count, $reset }
})
```

## Pinia Plugins

```ts
function myPlugin({ store }) {
  // Add property to every store
  store.createdAt = new Date()

  // Watch all state changes
  store.$subscribe((mutation) => {
    console.log(`[${store.$id}] changed`)
  })
}

const pinia = createPinia()
pinia.use(myPlugin)
```

### Persistence Plugin

```ts
// plugins/pinia-persist.client.ts
import piniaPluginPersistedstate from 'pinia-plugin-persistedstate'

export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.$pinia.use(piniaPluginPersistedstate)
})
```

```ts
export const useSettingsStore = defineStore('settings', () => {
  const theme = ref('light')
  return { theme }
}, { persist: true })  // Auto-syncs with localStorage
```

## Pinia in Nuxt

With `@pinia/nuxt`, stores are auto-imported:

```vue
<script setup>
// No import needed — auto-imported from stores/
const auth = useAuthStore()
</script>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Destructuring state loses reactivity | Use `storeToRefs()` |
| `$reset()` not working with setup syntax | Implement your own reset function |
| Calling `useStore()` outside setup | Only call inside `setup()`, plugins, or middleware |
| Circular store dependencies | Refactor shared logic into a composable |
