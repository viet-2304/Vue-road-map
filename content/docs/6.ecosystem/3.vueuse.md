# VueUse Utilities

VueUse is a collection of 200+ composable utilities for Vue. Think of it as Lodash for Vue's Composition API.

## Installation

```bash
npm install @vueuse/core
```

In Nuxt, use `@vueuse/nuxt` for auto-imports.

## Essential Composables

### useStorage — Reactive localStorage

```vue
<script setup>
import { useStorage } from '@vueuse/core'

// Syncs with localStorage automatically
const theme = useStorage('theme', 'light')  // key, default
const user = useStorage('user', { name: '', email: '' })
</script>

<template>
  <select v-model="theme">
    <option>light</option>
    <option>dark</option>
  </select>
</template>
```

### useMouse — Track Mouse Position

```vue
<script setup>
import { useMouse } from '@vueuse/core'

const { x, y } = useMouse()
</script>

<template>
  <p>Mouse: {{ x }}, {{ y }}</p>
</template>
```

### useClipboard — Copy to Clipboard

```vue
<script setup>
import { useClipboard } from '@vueuse/core'

const source = ref('Text to copy')
const { copy, copied } = useClipboard({ source })
</script>

<template>
  <button @click="copy()">
    {{ copied ? 'Copied!' : 'Copy' }}
  </button>
</template>
```

### useDark — Dark Mode

```vue
<script setup>
import { useDark, useToggle } from '@vueuse/core'

const isDark = useDark()
const toggleDark = useToggle(isDark)
</script>

<template>
  <button @click="toggleDark()">
    {{ isDark ? 'Light' : 'Dark' }} mode
  </button>
</template>
```

### useIntersectionObserver — Lazy Loading

```vue
<script setup>
import { useIntersectionObserver } from '@vueuse/core'

const target = ref<HTMLElement>()
const isVisible = ref(false)

useIntersectionObserver(target, ([{ isIntersecting }]) => {
  isVisible.value = isIntersecting
})
</script>

<template>
  <div ref="target">
    <ExpensiveComponent v-if="isVisible" />
  </div>
</template>
```

### useDebounce / useThrottle

```vue
<script setup>
import { useDebouncedRef } from '@vueuse/core'

const search = useDebouncedRef('', 300)  // 300ms delay
</script>

<template>
  <input v-model="search" placeholder="Search...">
</template>
```

### useFetch — Reactive HTTP

```vue
<script setup>
import { useFetch } from '@vueuse/core'

const { data, error, isFetching } = useFetch('https://api.example.com/users')
  .get()
  .json()
</script>

<template>
  <div v-if="isFetching">Loading...</div>
  <div v-else-if="error">Error: {{ error }}</div>
  <ul v-else>
    <li v-for="user in data" :key="user.id">{{ user.name }}</li>
  </ul>
</template>
```

### useEventListener — Auto-Cleanup

```vue
<script setup>
import { useEventListener } from '@vueuse/core'

// Automatically removed on unmount
useEventListener(window, 'resize', () => {
  console.log(window.innerWidth)
})

useEventListener(document, 'keydown', (e) => {
  if (e.key === 'Escape') closeModal()
})
</script>
```

## Patterns

### Combining Composables

```vue
<script setup>
import { useMouse, useWindowSize, computed } from '@vueuse/core'

const { x, y } = useMouse()
const { width, height } = useWindowSize()

const relativeX = computed(() => (x.value / width.value * 100).toFixed(1))
const relativeY = computed(() => (y.value / height.value * 100).toFixed(1))
</script>
```

### Creating Your Own VueUse-Style Composable

```ts
import { ref, onMounted, onUnmounted } from 'vue'

export function useOnlineStatus() {
  const isOnline = ref(navigator.onLine)

  function update() {
    isOnline.value = navigator.onLine
  }

  onMounted(() => {
    window.addEventListener('online', update)
    window.addEventListener('offline', update)
  })

  onUnmounted(() => {
    window.removeEventListener('online', update)
    window.removeEventListener('offline', update)
  })

  return { isOnline }
}
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| SSR errors with browser APIs | Most VueUse composables handle SSR — check docs |
| Importing from wrong package | `@vueuse/core` for most, `@vueuse/integrations` for third-party |
| Not cleaning up | VueUse handles cleanup automatically on unmount |
| Bundle size concerns | Tree-shakeable — only imports what you use |
