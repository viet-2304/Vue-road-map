# Form Validation

Form validation ensures user input meets requirements before submission. Vue ecosystem offers several approaches.

## Native HTML Validation

The simplest approach — browser-native:

```vue
<template>
  <form @submit.prevent="onSubmit">
    <input v-model="email" type="email" required>
    <input v-model="age" type="number" min="18" max="120" required>
    <input v-model="name" minlength="2" maxlength="50" required>
    <button type="submit">Submit</button>
  </form>
</template>
```

Limited styling and messaging control. For complex forms, use a validation library.

## Vee-Validate

The most popular Vue form validation library.

### Setup

```bash
npm install vee-validate @vee-validate/zod zod
```

### Basic Usage with Composition API

```vue
<script setup>
import { useForm, useField } from 'vee-validate'
import { z } from 'zod'
import { toTypedSchema } from '@vee-validate/zod'

const schema = toTypedSchema(z.object({
  email: z.string().email('Invalid email'),
  password: z.string().min(8, 'At least 8 characters'),
  name: z.string().min(2, 'Name is too short'),
}))

const { handleSubmit, errors } = useForm({ validationSchema: schema })

const { value: email } = useField('email')
const { value: password } = useField('password')
const { value: name } = useField('name')

const onSubmit = handleSubmit((values) => {
  console.log('Valid form:', values)
})
</script>

<template>
  <form @submit="onSubmit">
    <div>
      <input v-model="name" placeholder="Name">
      <span v-if="errors.name" class="error">{{ errors.name }}</span>
    </div>
    <div>
      <input v-model="email" type="email" placeholder="Email">
      <span v-if="errors.email" class="error">{{ errors.email }}</span>
    </div>
    <div>
      <input v-model="password" type="password" placeholder="Password">
      <span v-if="errors.password" class="error">{{ errors.password }}</span>
    </div>
    <button type="submit">Submit</button>
  </form>
</template>
```

### Form-Level Validation State

```vue
<script setup>
const { handleSubmit, errors, isSubmitting, meta } = useForm({
  validationSchema: schema
})

// meta.valid — all fields valid
// meta.dirty — any field changed
// meta.touched — any field interacted with
</script>

<template>
  <button type="submit" :disabled="!meta.valid || isSubmitting">
    {{ isSubmitting ? 'Submitting...' : 'Submit' }}
  </button>
</template>
```

## Zod Schema Validation

Zod provides type-safe schema validation that works standalone or with vee-validate:

```ts
import { z } from 'zod'

const userSchema = z.object({
  email: z.string().email(),
  age: z.number().min(18).max(120),
  role: z.enum(['admin', 'user', 'editor']),
  tags: z.array(z.string()).min(1, 'At least one tag'),
  address: z.object({
    street: z.string(),
    city: z.string(),
    zip: z.string().regex(/^\d{5}$/, 'Invalid ZIP'),
  }).optional(),
})

type User = z.infer<typeof userSchema>  // Auto-generated TypeScript type
```

### Custom Validation Rules

```ts
const passwordSchema = z.string()
  .min(8)
  .regex(/[A-Z]/, 'Must contain uppercase')
  .regex(/[0-9]/, 'Must contain number')
  .regex(/[^A-Za-z0-9]/, 'Must contain special character')

const registrationSchema = z.object({
  password: passwordSchema,
  confirmPassword: z.string(),
}).refine(data => data.password === data.confirmPassword, {
  message: 'Passwords must match',
  path: ['confirmPassword'],
})
```

## Manual Validation Pattern

For simple forms, no library needed:

```vue
<script setup>
const form = reactive({ email: '', name: '' })
const errors = reactive({ email: '', name: '' })

function validate(): boolean {
  errors.email = ''
  errors.name = ''

  if (!form.email.includes('@')) errors.email = 'Invalid email'
  if (form.name.length < 2) errors.name = 'Too short'

  return !errors.email && !errors.name
}

function onSubmit() {
  if (!validate()) return
  // Submit form...
}
</script>
```

## Async Validation

```ts
const schema = toTypedSchema(z.object({
  username: z.string().min(3).refine(async (val) => {
    const res = await fetch(`/api/check-username/${val}`)
    return (await res.json()).available
  }, 'Username already taken'),
}))
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Validating on every keystroke | Use `.lazy` modifier or debounce |
| Not showing errors at right time | Validate on blur, show on submit |
| Complex schemas becoming unreadable | Split into smaller reusable schemas |
| Server validation mismatch | Validate on both client and server |
