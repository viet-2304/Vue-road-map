# Vue Router

Vue Router is the official routing library for Vue.js, enabling single-page application navigation.

## Basic Setup

```ts
import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: '/', component: () => import('./pages/Home.vue') },
    { path: '/about', component: () => import('./pages/About.vue') },
  ]
})
```

In Nuxt, routing is file-based and automatic — no manual setup needed.

## Dynamic Routes

```ts
const routes = [
  // Dynamic segment
  { path: '/users/:id', component: UserProfile },

  // Multiple params
  { path: '/posts/:year/:month', component: PostArchive },

  // Optional param
  { path: '/docs/:section?', component: Docs },
]
```

Access params in components:

```vue
<script setup>
const route = useRoute()
const userId = computed(() => route.params.id)
</script>
```

## Nested Routes

```ts
const routes = [
  {
    path: '/dashboard',
    component: DashboardLayout,
    children: [
      { path: '', component: DashboardHome },       // /dashboard
      { path: 'settings', component: Settings },      // /dashboard/settings
      { path: 'profile', component: Profile },        // /dashboard/profile
    ]
  }
]
```

The parent component needs a `<RouterView>` for child routes:

```vue
<!-- DashboardLayout.vue -->
<template>
  <aside>Sidebar</aside>
  <main>
    <RouterView />  <!-- Child routes render here -->
  </main>
</template>
```

## Programmatic Navigation

```vue
<script setup>
const router = useRouter()

function goToUser(id: number) {
  router.push(`/users/${id}`)
  // Or with named routes:
  router.push({ name: 'user', params: { id } })
}

function goBack() {
  router.back()
}

function replaceRoute() {
  router.replace('/new-page')  // No history entry
}
</script>
```

## Named Routes

```ts
const routes = [
  { path: '/user/:id', name: 'user', component: UserProfile },
]
```

```vue
<template>
  <RouterLink :to="{ name: 'user', params: { id: 42 } }">Profile</RouterLink>
</template>
```

## Navigation Guards

### Global Guards

```ts
router.beforeEach((to, from) => {
  const auth = useAuthStore()

  if (to.meta.requiresAuth && !auth.isLoggedIn) {
    return { name: 'login', query: { redirect: to.fullPath } }
  }
})

router.afterEach((to) => {
  document.title = to.meta.title || 'My App'
})
```

### Per-Route Guards

```ts
const routes = [
  {
    path: '/admin',
    component: Admin,
    beforeEnter: (to, from) => {
      if (!isAdmin()) return '/unauthorized'
    }
  }
]
```

### In-Component Guards

```vue
<script setup>
onBeforeRouteLeave((to, from) => {
  if (hasUnsavedChanges.value) {
    return confirm('Discard changes?')
  }
})

onBeforeRouteUpdate((to, from) => {
  // Same component, different params (e.g., /user/1 → /user/2)
  fetchUser(to.params.id)
})
</script>
```

## Route Meta

```ts
const routes = [
  {
    path: '/admin',
    component: Admin,
    meta: { requiresAuth: true, title: 'Admin Panel' }
  }
]
```

## Lazy Loading Routes

```ts
const routes = [
  {
    path: '/dashboard',
    component: () => import('./pages/Dashboard.vue'),  // Code-split
  }
]
```

In Nuxt, all page components are automatically lazy-loaded.

## Catch-All / 404

```ts
const routes = [
  // ... other routes
  { path: '/:pathMatch(.*)*', name: 'not-found', component: NotFound }
]
```

## Nuxt File-Based Routing

```
pages/
  index.vue          → /
  about.vue          → /about
  users/
    index.vue        → /users
    [id].vue         → /users/:id
  posts/
    [...slug].vue    → /posts/* (catch-all)
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Component not updating on param change | Use `watch(() => route.params.id)` or add `:key="route.path"` |
| Guards returning nothing | Return `true`, `false`, or a route to navigate to |
| Infinite redirect loops | Check conditions carefully in `beforeEach` |
| Missing `<RouterView>` for nested routes | Parent layout must include it |
