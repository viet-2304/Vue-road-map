# Vitest Fundamentals

Vitest is a Vite-native test framework — fast, ESM-first, and compatible with the Jest API.

## Setup

```bash
npm install -D vitest
```

### Configuration

```ts
// vitest.config.ts
import { defineConfig } from 'vitest/config'

export default defineConfig({
  test: {
    globals: true,           // No need to import describe/it/expect
    environment: 'happy-dom', // or 'jsdom' for DOM testing
  }
})
```

Or add to `vite.config.ts`:

```ts
export default defineConfig({
  test: {
    globals: true,
    environment: 'happy-dom',
  }
})
```

### Package.json Scripts

```json
{
  "scripts": {
    "test": "vitest",
    "test:run": "vitest run",
    "test:coverage": "vitest run --coverage"
  }
}
```

## Writing Tests

### Basic Structure

```ts
// math.test.ts
import { add, multiply } from './math'

describe('Math utilities', () => {
  it('adds two numbers', () => {
    expect(add(2, 3)).toBe(5)
  })

  it('multiplies two numbers', () => {
    expect(multiply(3, 4)).toBe(12)
  })
})
```

### test vs it

`test` and `it` are aliases — use whichever reads better:

```ts
test('returns true for valid email', () => { ... })
it('should return true for valid email', () => { ... })
```

## Matchers

### Common Matchers

```ts
// Equality
expect(value).toBe(5)              // Strict ===
expect(obj).toEqual({ a: 1 })      // Deep equality
expect(obj).toStrictEqual({ a: 1 }) // Deep + type equality

// Truthiness
expect(value).toBeTruthy()
expect(value).toBeFalsy()
expect(value).toBeNull()
expect(value).toBeUndefined()
expect(value).toBeDefined()

// Numbers
expect(value).toBeGreaterThan(3)
expect(value).toBeLessThanOrEqual(10)
expect(0.1 + 0.2).toBeCloseTo(0.3)

// Strings
expect(str).toMatch(/regex/)
expect(str).toContain('substring')

// Arrays
expect(arr).toContain(item)
expect(arr).toHaveLength(3)

// Objects
expect(obj).toHaveProperty('key')
expect(obj).toHaveProperty('nested.key', 'value')

// Exceptions
expect(() => fn()).toThrow()
expect(() => fn()).toThrow('error message')
expect(() => fn()).toThrowError(TypeError)
```

### Negation

```ts
expect(value).not.toBe(5)
expect(arr).not.toContain(item)
```

## Setup and Teardown

```ts
describe('Database tests', () => {
  beforeAll(async () => {
    await db.connect()
  })

  afterAll(async () => {
    await db.disconnect()
  })

  beforeEach(() => {
    db.clear()
  })

  afterEach(() => {
    // Cleanup after each test
  })

  it('inserts a record', () => { ... })
})
```

## Async Tests

```ts
it('fetches user data', async () => {
  const user = await fetchUser(1)
  expect(user.name).toBe('Alice')
})

it('rejects with an error', async () => {
  await expect(fetchUser(-1)).rejects.toThrow('Not found')
})
```

## Test Organization

### Nested describe

```ts
describe('UserService', () => {
  describe('create', () => {
    it('creates a user with valid data', () => { ... })
    it('throws on invalid email', () => { ... })
  })

  describe('delete', () => {
    it('deletes an existing user', () => { ... })
    it('throws on non-existent user', () => { ... })
  })
})
```

### Skip and Focus

```ts
it.skip('not ready yet', () => { ... })     // Skip this test
it.only('focus on this', () => { ... })     // Run only this test
describe.skip('whole suite', () => { ... }) // Skip suite
it.todo('implement later')                  // Placeholder
```

## CLI Usage

```bash
vitest              # Watch mode (default)
vitest run          # Single run
vitest run math     # Run tests matching "math"
vitest --reporter=verbose  # Detailed output
vitest --coverage   # With coverage report
```

## Watch Mode

Vitest runs in watch mode by default — only re-runs tests affected by changed files. This is powered by Vite's HMR.

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| `describe`/`it` not found | Enable `globals: true` in config or import from `vitest` |
| DOM APIs not available | Set `environment: 'happy-dom'` or `'jsdom'` |
| Tests passing in isolation but failing together | Check for shared state — use `beforeEach` to reset |
| Async test timing out | Increase timeout: `it('test', async () => { ... }, 10000)` |
