# Mocking & Advanced Patterns

Advanced Vitest techniques — module mocking, spies, fake timers, snapshot testing, and code coverage.

## vi.mock — Module Mocking

Replace entire modules with controlled implementations:

```ts
// Mock before imports are resolved
vi.mock('./api', () => ({
  fetchUsers: vi.fn().mockResolvedValue([{ id: 1, name: 'Alice' }]),
  createUser: vi.fn().mockResolvedValue({ id: 2, name: 'Bob' }),
}))

import { fetchUsers, createUser } from './api'

it('uses mocked fetchUsers', async () => {
  const users = await fetchUsers()
  expect(users).toEqual([{ id: 1, name: 'Alice' }])
})
```

### Partial Mocking

Keep real implementations for some exports:

```ts
vi.mock('./utils', async (importOriginal) => {
  const actual = await importOriginal<typeof import('./utils')>()
  return {
    ...actual,                    // Keep all real exports
    formatDate: vi.fn(() => '2026-01-01'),  // Override just this one
  }
})
```

### Resetting Mocks

```ts
afterEach(() => {
  vi.restoreAllMocks()  // Restore original implementations
})

// Or per mock:
const mockFn = vi.fn()
mockFn.mockReset()    // Clears calls + implementation
mockFn.mockRestore()  // Restores original (if spy)
mockFn.mockClear()    // Clears calls only
```

## vi.spyOn — Spying on Methods

Watch method calls without replacing them:

```ts
it('calls console.warn on invalid input', () => {
  const spy = vi.spyOn(console, 'warn').mockImplementation(() => {})

  validate('')
  expect(spy).toHaveBeenCalledWith('Input is empty')

  spy.mockRestore()
})
```

### Spy on Object Methods

```ts
const cart = {
  items: [],
  total() { return this.items.reduce((sum, i) => sum + i.price, 0) },
}

const spy = vi.spyOn(cart, 'total')
cart.total()
expect(spy).toHaveBeenCalledOnce()
expect(spy).toHaveReturnedWith(0)
```

## Fake Timers

Control `setTimeout`, `setInterval`, and `Date`:

```ts
describe('Debounce', () => {
  beforeEach(() => vi.useFakeTimers())
  afterEach(() => vi.useRealTimers())

  it('delays execution', () => {
    const fn = vi.fn()
    const debounced = debounce(fn, 300)

    debounced()
    expect(fn).not.toHaveBeenCalled()

    vi.advanceTimersByTime(300)
    expect(fn).toHaveBeenCalledOnce()
  })

  it('resets timer on repeated calls', () => {
    const fn = vi.fn()
    const debounced = debounce(fn, 300)

    debounced()
    vi.advanceTimersByTime(200)
    debounced()                       // Resets the timer
    vi.advanceTimersByTime(200)
    expect(fn).not.toHaveBeenCalled() // Only 200ms since last call

    vi.advanceTimersByTime(100)
    expect(fn).toHaveBeenCalledOnce()
  })
})
```

### Controlling Date

```ts
it('uses a fixed date', () => {
  vi.setSystemTime(new Date('2026-06-15'))

  expect(new Date().toISOString()).toContain('2026-06-15')

  vi.useRealTimers()
})
```

### Timer Utilities

```ts
vi.advanceTimersByTime(1000)   // Advance by 1 second
vi.advanceTimersToNextTimer()  // Jump to next scheduled timer
vi.runAllTimers()              // Run all pending timers
vi.runOnlyPendingTimers()      // Run currently pending (not new ones)
vi.getTimerCount()             // Number of pending timers
```

## Mocking Global APIs

### fetch

```ts
global.fetch = vi.fn().mockResolvedValue({
  ok: true,
  json: () => Promise.resolve({ data: 'test' }),
})

// Or with different responses per call:
global.fetch = vi.fn()
  .mockResolvedValueOnce({ ok: true, json: () => Promise.resolve({ id: 1 }) })
  .mockRejectedValueOnce(new Error('Network error'))
```

### localStorage

```ts
const store: Record<string, string> = {}

vi.stubGlobal('localStorage', {
  getItem: vi.fn((key: string) => store[key] ?? null),
  setItem: vi.fn((key: string, val: string) => { store[key] = val }),
  removeItem: vi.fn((key: string) => { delete store[key] }),
  clear: vi.fn(() => Object.keys(store).forEach(k => delete store[k])),
})
```

### window.matchMedia

```ts
vi.stubGlobal('matchMedia', vi.fn().mockImplementation((query) => ({
  matches: query === '(prefers-color-scheme: dark)',
  media: query,
  addEventListener: vi.fn(),
  removeEventListener: vi.fn(),
})))
```

## Snapshot Testing

Capture output and compare against saved snapshots:

```ts
it('renders correctly', () => {
  const wrapper = mount(UserCard, {
    props: { name: 'Alice', role: 'admin' },
  })

  expect(wrapper.html()).toMatchSnapshot()
})
```

First run creates a `.snap` file. Subsequent runs compare against it.

### Inline Snapshots

```ts
it('formats greeting', () => {
  expect(formatGreeting('Alice')).toMatchInlineSnapshot(`"Hello, Alice!"`)
})
```

The expected value is written directly into the test file by Vitest.

### Updating Snapshots

```bash
vitest --update     # Update all snapshots
vitest -u           # Short form
```

### When to Use Snapshots

| Good For | Bad For |
|---|---|
| HTML output of small components | Large, frequently changing templates |
| Serialized data structures | Dynamic content (timestamps, IDs) |
| Error messages | Entire page renders |

## Code Coverage

### Setup

```bash
npm install -D @vitest/coverage-v8
```

```ts
// vitest.config.ts
export default defineConfig({
  test: {
    coverage: {
      provider: 'v8',
      reporter: ['text', 'html', 'lcov'],
      include: ['src/**/*.{ts,vue}'],
      exclude: ['**/*.test.ts', '**/*.spec.ts'],
    },
  },
})
```

### Running Coverage

```bash
vitest run --coverage
```

### Coverage Thresholds

```ts
coverage: {
  thresholds: {
    lines: 80,
    functions: 80,
    branches: 75,
    statements: 80,
  },
}
```

Build fails if coverage drops below thresholds.

## Parameterized Tests

Run the same test with different inputs:

```ts
it.each([
  { input: 'hello@test.com', expected: true },
  { input: 'invalid', expected: false },
  { input: '', expected: false },
  { input: 'a@b.co', expected: true },
])('validates "$input" as $expected', ({ input, expected }) => {
  expect(isValidEmail(input)).toBe(expected)
})
```

### Table Syntax

```ts
it.each`
  a    | b    | expected
  ${1} | ${2} | ${3}
  ${3} | ${4} | ${7}
`('$a + $b = $expected', ({ a, b, expected }) => {
  expect(add(a, b)).toBe(expected)
})
```

## Test Fixtures

Share setup logic across test files:

```ts
// fixtures/db.ts
import { test as base } from 'vitest'

export const test = base.extend<{ db: Database }>({
  db: async ({}, use) => {
    const db = await createTestDB()
    await use(db)
    await db.cleanup()
  },
})
```

```ts
import { test } from './fixtures/db'

test('inserts a record', async ({ db }) => {
  await db.insert({ name: 'Alice' })
  const result = await db.findAll()
  expect(result).toHaveLength(1)
})
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| `vi.mock` not hoisted | `vi.mock` is auto-hoisted to top of file — don't put it inside `describe`/`it` |
| Spy not restoring after test | Use `vi.restoreAllMocks()` in `afterEach` |
| Fake timers breaking async tests | Call `vi.useRealTimers()` in `afterEach` |
| Snapshots too large and fragile | Snapshot small, stable pieces; avoid full-page snapshots |
| Coverage not reflecting reality | High coverage ≠ good tests — focus on meaningful assertions |
