# Render Functions & JSX

Templates are compiled to render functions. You can write render functions directly for more programmatic control.

## The h() Function

`h()` (hyperscript) creates virtual DOM nodes (VNodes):

```ts
import { h } from 'vue'

// h(type, props, children)
h('div', { class: 'container' }, 'Hello World')
h('div', { id: 'app' }, [
  h('h1', 'Title'),
  h('p', 'Content')
])
```

## Render Function Component

```ts
import { h, ref } from 'vue'

export default {
  setup() {
    const count = ref(0)

    return () => h('div', [
      h('p', `Count: ${count.value}`),
      h('button', { onClick: () => count.value++ }, 'Increment')
    ])
  }
}
```

With `<script setup>`, you can't return a render function directly. Use a separate component file or `defineComponent`.

## VNode Props

```ts
h('div', {
  // Attributes
  id: 'my-div',
  class: ['foo', { bar: true }],
  style: { color: 'red' },

  // Events (prefix with on)
  onClick: (e) => console.log(e),
  onMouseenter: () => {},

  // DOM properties
  innerHTML: '<b>bold</b>',
})
```

## Rendering Components

```ts
import MyComponent from './MyComponent.vue'

h(MyComponent, {
  modelValue: value,
  'onUpdate:modelValue': (val) => { value = val },
  title: 'Hello',
  onClose: () => {}
})
```

## Slots in Render Functions

### Passing Slots

```ts
h(MyComponent, null, {
  // Named slots as functions
  default: () => h('p', 'Default slot content'),
  header: () => h('h1', 'Header'),
  footer: ({ count }) => h('p', `Footer: ${count}`),  // Scoped slot
})
```

### Receiving Slots

```ts
export default {
  setup(props, { slots }) {
    return () => h('div', [
      h('header', slots.header?.()),
      h('main', slots.default?.()),
      h('footer', slots.footer?.({ count: 42 })),
    ])
  }
}
```

## Conditional and List Rendering

No special syntax needed — use JavaScript directly:

```ts
export default {
  setup() {
    const items = ref(['A', 'B', 'C'])
    const show = ref(true)

    return () => h('div', [
      // v-if equivalent
      show.value ? h('p', 'Visible') : null,

      // v-for equivalent
      h('ul', items.value.map(item =>
        h('li', { key: item }, item)
      ))
    ])
  }
}
```

## Functional Components

Stateless components defined as plain functions:

```ts
function Heading(props: { level: number }, { slots }: any) {
  const tag = `h${props.level}`
  return h(tag, slots.default?.())
}

// Declare props
Heading.props = { level: { type: Number, required: true } }
```

Usage: `h(Heading, { level: 2 }, () => 'My Title')`

## JSX in Vue

JSX provides a more readable syntax for render functions. Requires `@vitejs/plugin-vue-jsx`:

```tsx
// components/Counter.tsx
import { defineComponent, ref } from 'vue'

export default defineComponent({
  setup() {
    const count = ref(0)

    return () => (
      <div>
        <p>Count: {count.value}</p>
        <button onClick={() => count.value++}>+1</button>

        {count.value > 5 && <p>High count!</p>}

        <ul>
          {[1, 2, 3].map(n => <li key={n}>Item {n}</li>)}
        </ul>
      </div>
    )
  }
})
```

### JSX Slots

```tsx
<MyComponent>
  {{
    default: () => <p>Default</p>,
    header: () => <h1>Header</h1>,
    footer: ({ count }) => <p>Count: {count}</p>,
  }}
</MyComponent>
```

## When to Use Render Functions

| Use Case | Template | Render Function |
|---|---|---|
| Standard UI | Preferred | Overkill |
| Dynamic tag names | `<component :is>` | `h(tag)` — cleaner |
| Highly dynamic children | Awkward | Natural |
| Component libraries | Limited | Full control |
| Programmatic component creation | Not possible | Required |

**Rule of thumb:** Use templates for 95% of cases. Render functions shine for dynamic, programmatic component creation.

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Forgetting `key` in lists | Always provide `key` in `h()` props for `v-for` equivalents |
| Events named wrong | Use `onClick`, not `click` or `@click` |
| VNodes are not reusable | Create new VNodes each render, don't store references |
| Missing reactivity | Return a function from `setup`, not a static VNode |
