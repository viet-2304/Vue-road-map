# Custom Directives

Custom directives let you apply low-level DOM manipulations as reusable attributes.

## Basic Directive

A directive is an object with lifecycle hooks:

```vue
<script setup>
const vFocus = {
  mounted: (el: HTMLElement) => el.focus()
}
</script>

<template>
  <input v-focus placeholder="I'm auto-focused">
</template>
```

In `<script setup>`, any `camelCase` variable starting with `v` is available as a directive.

## Directive Hooks

```ts
const vMyDirective = {
  // Called before element's attributes are applied
  created(el, binding, vnode) {},
  // Called when element is inserted into DOM
  mounted(el, binding, vnode) {},
  // Called before the parent component updates
  beforeUpdate(el, binding, vnode, prevVnode) {},
  // Called after parent component has updated
  updated(el, binding, vnode, prevVnode) {},
  // Called before parent component unmounts
  beforeUnmount(el, binding, vnode) {},
  // Called when parent component unmounts
  unmounted(el, binding, vnode) {},
}
```

### The binding Object

```ts
interface DirectiveBinding {
  value: any        // Current value: v-dir="value"
  oldValue: any     // Previous value (in updated)
  arg: string       // Argument: v-dir:arg
  modifiers: object // Modifiers: v-dir.mod1.mod2
  instance: object  // Component instance
  dir: object       // The directive definition
}
```

## Arguments and Modifiers

```vue
<script setup>
const vColor = {
  mounted(el: HTMLElement, binding: any) {
    const color = binding.value || 'red'
    const property = binding.arg || 'color'  // v-color:background
    el.style[property] = color
  },
  updated(el: HTMLElement, binding: any) {
    const property = binding.arg || 'color'
    el.style[property] = binding.value
  }
}
</script>

<template>
  <p v-color="'blue'">Blue text</p>
  <p v-color:background="'yellow'">Yellow background</p>
</template>
```

### Using Modifiers

```vue
<script setup>
const vTooltip = {
  mounted(el: HTMLElement, binding: any) {
    const position = binding.modifiers.top ? 'top'
      : binding.modifiers.bottom ? 'bottom'
      : 'right'

    el.setAttribute('title', binding.value)
    el.dataset.tooltipPosition = position
  }
}
</script>

<template>
  <button v-tooltip.top="'Save changes'">Save</button>
  <button v-tooltip.bottom="'Delete item'">Delete</button>
</template>
```

## Function Shorthand

When you only need `mounted` and `updated` with the same logic:

```vue
<script setup>
const vColor = (el: HTMLElement, binding: any) => {
  el.style.color = binding.value
}
</script>

<template>
  <p v-color="'green'">Green text</p>
</template>
```

## Practical Examples

### v-click-outside

```ts
const vClickOutside = {
  mounted(el: HTMLElement, binding: any) {
    el._clickOutside = (event: Event) => {
      if (!el.contains(event.target as Node)) {
        binding.value(event)
      }
    }
    document.addEventListener('click', el._clickOutside)
  },
  unmounted(el: HTMLElement) {
    document.removeEventListener('click', el._clickOutside)
  }
}
```

```vue
<template>
  <div v-click-outside="closeDropdown" class="dropdown">
    <button @click="open = !open">Menu</button>
    <ul v-show="open">...</ul>
  </div>
</template>
```

### v-intersection (Lazy Loading)

```ts
const vIntersection = {
  mounted(el: HTMLElement, binding: any) {
    const observer = new IntersectionObserver(([entry]) => {
      if (entry.isIntersecting) {
        binding.value(el)
        observer.disconnect()
      }
    })
    observer.observe(el)
    el._observer = observer
  },
  unmounted(el: HTMLElement) {
    el._observer?.disconnect()
  }
}
```

## Global Registration

Register in Nuxt via a plugin:

```ts
// plugins/directives.ts
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.vueApp.directive('focus', {
    mounted: (el) => el.focus()
  })
})
```

## When to Use Directives vs Composables

| Use Case | Directive | Composable |
|---|---|---|
| Direct DOM manipulation | Yes | No |
| Reactive state | Limited | Yes |
| Reusable logic without DOM | No | Yes |
| Third-party library binding | Yes | Maybe |

**Prefer composables** for most cases. Use directives only when you need direct DOM access that can't be achieved with template refs.

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Not cleaning up in `unmounted` | Always remove event listeners, observers |
| Accessing component state | Use `binding.instance` carefully |
| Complex logic in directives | Extract to a composable instead |
| Forgetting `updated` hook | Value changes won't be reflected |
