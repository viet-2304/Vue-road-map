# Teleport

Vue components render within a nested DOM hierarchy that mirrors the component tree. But sometimes you need an element -- like a modal, tooltip, or toast notification -- to appear at a completely different place in the DOM. The `<Teleport>` component solves this by "teleporting" its content to a target element outside the component's DOM subtree.

## Why Teleport Exists

Consider a modal dialog built inside a deeply nested component. Even though the modal logically belongs to that component, it should render at the top level of the `<body>` to avoid CSS stacking context issues (`z-index`, `overflow: hidden`, `transform`). Before Teleport, developers had to manually manage DOM portals. Vue's built-in `<Teleport>` handles this declaratively.

## Basic Usage

The `<Teleport>` component accepts a `to` prop that specifies where its children should be rendered in the DOM.

```vue
<script setup>
import { ref } from 'vue'

const showModal = ref(false)
</script>

<template>
  <button @click="showModal = true">Open Modal</button>

  <Teleport to="body">
    <div v-if="showModal" class="modal-overlay">
      <div class="modal">
        <h2>Hello from Teleport!</h2>
        <p>This renders directly in the body element.</p>
        <button @click="showModal = false">Close</button>
      </div>
    </div>
  </Teleport>
</template>

<style scoped>
.modal-overlay {
  position: fixed;
  inset: 0;
  background: rgba(0, 0, 0, 0.5);
  display: flex;
  align-items: center;
  justify-content: center;
}

.modal {
  background: white;
  padding: 2rem;
  border-radius: 8px;
  min-width: 300px;
}
</style>
```

Even though this component is nested deep in the component tree, the modal overlay renders as a direct child of `<body>`.

## The `to` Prop

The `to` prop accepts a CSS selector string or an actual DOM element reference.

### CSS Selector

```vue
<!-- Renders into the element matching the selector -->
<Teleport to="#modals">
  <div class="modal">...</div>
</Teleport>

<Teleport to=".notification-area">
  <div class="toast">...</div>
</Teleport>
```

Make sure the target element exists in your `index.html` or app layout:

```html
<!-- index.html -->
<body>
  <div id="app"></div>
  <div id="modals"></div>
  <div class="notification-area"></div>
</body>
```

### DOM Element Reference

You can also pass a DOM element directly:

```vue
<script setup>
import { ref, onMounted } from 'vue'

const targetEl = ref(null)

onMounted(() => {
  targetEl.value = document.getElementById('modals')
})
</script>

<template>
  <Teleport v-if="targetEl" :to="targetEl">
    <div class="modal">Teleported content</div>
  </Teleport>
</template>
```

## The `disabled` Prop

Sometimes you want to conditionally disable teleportation -- for example, rendering content inline on mobile but teleported on desktop.

```vue
<script setup>
import { ref } from 'vue'

const isMobile = ref(window.innerWidth < 768)
</script>

<template>
  <Teleport to="#sidebar" :disabled="isMobile">
    <nav class="navigation">
      <ul>
        <li>Home</li>
        <li>About</li>
        <li>Contact</li>
      </ul>
    </nav>
  </Teleport>
</template>
```

When `disabled` is `true`, the content renders in its natural position within the component tree. When `false`, it teleports to the target.

## Multiple Teleports to the Same Target

Multiple `<Teleport>` components can send content to the same target. The content appends in order.

```vue
<template>
  <Teleport to="#notifications">
    <div class="toast toast-info">Info message</div>
  </Teleport>

  <Teleport to="#notifications">
    <div class="toast toast-success">Success message</div>
  </Teleport>
</template>
```

Both toasts appear inside `#notifications`, in the order they are declared. This is especially useful for notification systems where multiple components can independently push messages.

## Teleport with Transitions

Teleported content works seamlessly with Vue transitions:

```vue
<script setup>
import { ref } from 'vue'

const showModal = ref(false)
</script>

<template>
  <button @click="showModal = true">Open</button>

  <Teleport to="body">
    <Transition name="fade">
      <div v-if="showModal" class="modal-overlay" @click.self="showModal = false">
        <div class="modal">
          <p>Animated modal!</p>
          <button @click="showModal = false">Close</button>
        </div>
      </div>
    </Transition>
  </Teleport>
</template>

<style>
.fade-enter-active,
.fade-leave-active {
  transition: opacity 0.3s ease;
}

.fade-enter-from,
.fade-leave-to {
  opacity: 0;
}
</style>
```

## Practical Use Cases

### Tooltip Component

```vue
<!-- Tooltip.vue -->
<script setup>
import { ref } from 'vue'

const props = defineProps({
  text: String
})

const show = ref(false)
const position = ref({ top: 0, left: 0 })
const triggerRef = ref(null)

function onMouseEnter() {
  const rect = triggerRef.value.getBoundingClientRect()
  position.value = {
    top: rect.top - 8 + window.scrollY,
    left: rect.left + rect.width / 2
  }
  show.value = true
}

function onMouseLeave() {
  show.value = false
}
</script>

<template>
  <span
    ref="triggerRef"
    @mouseenter="onMouseEnter"
    @mouseleave="onMouseLeave"
  >
    <slot />
  </span>

  <Teleport to="body">
    <div
      v-if="show"
      class="tooltip"
      :style="{
        top: position.top + 'px',
        left: position.left + 'px'
      }"
    >
      {{ text }}
    </div>
  </Teleport>
</template>
```

### Toast Notification System

```vue
<!-- ToastContainer.vue -->
<script setup>
import { ref, provide } from 'vue'

const toasts = ref([])
let nextId = 0

function addToast(message, type = 'info') {
  const id = nextId++
  toasts.value.push({ id, message, type })
  setTimeout(() => removeToast(id), 3000)
}

function removeToast(id) {
  toasts.value = toasts.value.filter(t => t.id !== id)
}

provide('toast', { addToast })
</script>

<template>
  <slot />

  <Teleport to="body">
    <div class="toast-container">
      <TransitionGroup name="slide">
        <div
          v-for="toast in toasts"
          :key="toast.id"
          :class="['toast', `toast-${toast.type}`]"
        >
          {{ toast.message }}
        </div>
      </TransitionGroup>
    </div>
  </Teleport>
</template>
```

```vue
<!-- Any descendant component -->
<script setup>
import { inject } from 'vue'

const { addToast } = inject('toast')

function handleSave() {
  addToast('Item saved successfully!', 'success')
}
</script>
```

## Teleport in SSR (Nuxt)

When using server-side rendering, the teleport target must exist when the component renders on the client. In Nuxt, you can wrap teleported content in `<ClientOnly>` or ensure the target element is part of your HTML layout.

```vue
<ClientOnly>
  <Teleport to="#modals">
    <Modal v-if="showModal" />
  </Teleport>
</ClientOnly>
```

Alternatively, Nuxt 3 and Vue 3.5+ handle teleport hydration more gracefully, but you should still verify the target element exists before teleporting.

## Common Pitfalls

1. **Target element does not exist.** If the selector in `to` does not match any element, Vue will log a warning and the content will not render. Always ensure the target element exists before the Teleport mounts.

2. **CSS scoped styles.** Scoped styles from the parent component still apply to teleported content because scoping is handled at compile time. However, styles relying on DOM hierarchy (like descendant selectors from a wrapper) will break.

3. **Event bubbling.** Even though teleported content lives elsewhere in the DOM, Vue events (`$emit`) still bubble through the **component tree**, not the DOM tree. A teleported child can still emit events to its logical parent.

4. **Reactivity is preserved.** Teleported content maintains full reactivity and connection to its parent component. Props, data, computed properties, and event handlers all work as expected.

## Summary

- `<Teleport>` renders content at a different place in the DOM while keeping it logically part of the component tree.
- The `to` prop accepts a CSS selector or DOM element reference.
- The `disabled` prop conditionally disables teleportation.
- Multiple teleports can target the same element and content appends in order.
- Teleported content preserves reactivity, events, and scoped styles.
- Common use cases include modals, tooltips, and notification systems.
