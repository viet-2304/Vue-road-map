# Forms Handling

Vue's `v-model` makes form handling straightforward. This lesson covers all input types, modifiers, and common form patterns.

## Text Inputs

```vue
<script setup>
const name = ref('')
const bio = ref('')
</script>

<template>
  <input v-model="name" placeholder="Name">
  <textarea v-model="bio" placeholder="Bio"></textarea>
  <p>Name: {{ name }}, Bio: {{ bio }}</p>
</template>
```

## v-model Modifiers

### .lazy

Syncs on `change` event instead of `input` (updates on blur):

```vue
<input v-model.lazy="message">
```

### .number

Automatically parses input as a number:

```vue
<input v-model.number="age" type="number">
```

### .trim

Strips leading/trailing whitespace:

```vue
<input v-model.trim="username">
```

### Combining Modifiers

```vue
<input v-model.lazy.trim="email" type="email">
```

## Checkboxes

### Single Checkbox (boolean)

```vue
<script setup>
const agreed = ref(false)
</script>

<template>
  <label>
    <input type="checkbox" v-model="agreed">
    I agree to the terms
  </label>
  <p>Agreed: {{ agreed }}</p>
</template>
```

### Multiple Checkboxes (array)

```vue
<script setup>
const selectedFruits = ref<string[]>([])
</script>

<template>
  <label><input type="checkbox" value="apple" v-model="selectedFruits"> Apple</label>
  <label><input type="checkbox" value="banana" v-model="selectedFruits"> Banana</label>
  <label><input type="checkbox" value="cherry" v-model="selectedFruits"> Cherry</label>
  <p>Selected: {{ selectedFruits }}</p>
</template>
```

## Select

### Single Select

```vue
<script setup>
const selected = ref('')
</script>

<template>
  <select v-model="selected">
    <option disabled value="">Choose one</option>
    <option>Vue</option>
    <option>React</option>
    <option>Angular</option>
  </select>
</template>
```

### Multiple Select

```vue
<script setup>
const selectedFrameworks = ref<string[]>([])
</script>

<template>
  <select v-model="selectedFrameworks" multiple>
    <option>Vue</option>
    <option>React</option>
    <option>Angular</option>
  </select>
</template>
```

### Dynamic Options

```vue
<script setup>
const options = ref([
  { value: 'vue', label: 'Vue.js' },
  { value: 'react', label: 'React' },
  { value: 'angular', label: 'Angular' },
])
const selected = ref('vue')
</script>

<template>
  <select v-model="selected">
    <option v-for="opt in options" :key="opt.value" :value="opt.value">
      {{ opt.label }}
    </option>
  </select>
</template>
```

## Form Submission

```vue
<script setup>
const form = reactive({
  name: '',
  email: '',
  message: '',
})
const submitting = ref(false)

async function handleSubmit() {
  submitting.value = true
  try {
    await fetch('/api/contact', {
      method: 'POST',
      body: JSON.stringify(form),
    })
    alert('Sent!')
  } finally {
    submitting.value = false
  }
}
</script>

<template>
  <form @submit.prevent="handleSubmit">
    <input v-model="form.name" required placeholder="Name">
    <input v-model="form.email" type="email" required placeholder="Email">
    <textarea v-model="form.message" required placeholder="Message"></textarea>
    <button type="submit" :disabled="submitting">
      {{ submitting ? 'Sending...' : 'Send' }}
    </button>
  </form>
</template>
```

## Form Reset Pattern

```vue
<script setup>
const initialState = { name: '', email: '', role: 'user' }
const form = reactive({ ...initialState })

function resetForm() {
  Object.assign(form, initialState)
}
</script>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| `<textarea>{{ text }}</textarea>` | Use `v-model` instead of interpolation |
| Select without disabled placeholder | Add `<option disabled value="">` as first option |
| `.number` returns string for empty input | Check for empty string before using value |
| Forgetting `@submit.prevent` | Form will cause a full page reload without it |
