# Transitions & Animations

Vue provides `<Transition>` and `<TransitionGroup>` components for applying enter/leave animations.

## The Transition Component

Wraps a single element or component that toggles with `v-if`, `v-show`, or dynamic components:

```vue
<script setup>
const show = ref(true)
</script>

<template>
  <button @click="show = !show">Toggle</button>
  <Transition name="fade">
    <p v-if="show">Hello!</p>
  </Transition>
</template>

<style>
.fade-enter-active, .fade-leave-active {
  transition: opacity 0.3s ease;
}
.fade-enter-from, .fade-leave-to {
  opacity: 0;
}
</style>
```

## CSS Transition Classes

Vue applies six classes during enter/leave:

| Class | When |
|---|---|
| `v-enter-from` | Start state of enter |
| `v-enter-active` | Active state during enter |
| `v-enter-to` | End state of enter |
| `v-leave-from` | Start state of leave |
| `v-leave-active` | Active state during leave |
| `v-leave-to` | End state of leave |

With `name="fade"`, classes become `fade-enter-from`, `fade-leave-active`, etc.

### Slide Transition Example

```css
.slide-enter-active, .slide-leave-active {
  transition: transform 0.3s ease, opacity 0.3s ease;
}
.slide-enter-from {
  transform: translateX(-30px);
  opacity: 0;
}
.slide-leave-to {
  transform: translateX(30px);
  opacity: 0;
}
```

## CSS Animations

Use CSS `@keyframes` for more complex animations:

```vue
<template>
  <Transition name="bounce">
    <p v-if="show">Bouncy!</p>
  </Transition>
</template>

<style>
.bounce-enter-active {
  animation: bounce-in 0.5s;
}
.bounce-leave-active {
  animation: bounce-in 0.5s reverse;
}
@keyframes bounce-in {
  0% { transform: scale(0); }
  50% { transform: scale(1.25); }
  100% { transform: scale(1); }
}
</style>
```

## JavaScript Hooks

For programmatic animations (GSAP, Web Animations API):

```vue
<template>
  <Transition
    @before-enter="onBeforeEnter"
    @enter="onEnter"
    @after-enter="onAfterEnter"
    @leave="onLeave"
    :css="false"
  >
    <div v-if="show">Animated</div>
  </Transition>
</template>

<script setup>
function onEnter(el: Element, done: () => void) {
  el.animate(
    [{ opacity: 0, transform: 'scale(0.5)' }, { opacity: 1, transform: 'scale(1)' }],
    { duration: 300, easing: 'ease-out' }
  ).onfinish = done
}

function onLeave(el: Element, done: () => void) {
  el.animate(
    [{ opacity: 1 }, { opacity: 0 }],
    { duration: 200 }
  ).onfinish = done
}
</script>
```

Set `:css="false"` to skip CSS detection and rely entirely on JS hooks.

## Transition Modes

Control the timing of enter/leave transitions:

```vue
<template>
  <!-- New element enters AFTER current leaves -->
  <Transition name="fade" mode="out-in">
    <component :is="currentView" :key="currentView" />
  </Transition>
</template>
```

| Mode | Behavior |
|---|---|
| (default) | Enter and leave happen simultaneously |
| `out-in` | Current leaves first, then new enters |
| `in-out` | New enters first, then current leaves |

`out-in` is the most commonly used mode.

## TransitionGroup

Animates lists of elements rendered with `v-for`:

```vue
<script setup>
const items = ref([1, 2, 3])

function addItem() {
  items.value.push(items.value.length + 1)
}
function removeItem(item: number) {
  items.value = items.value.filter(i => i !== item)
}
</script>

<template>
  <button @click="addItem">Add</button>
  <TransitionGroup name="list" tag="ul">
    <li v-for="item in items" :key="item" @click="removeItem(item)">
      Item {{ item }}
    </li>
  </TransitionGroup>
</template>

<style>
.list-enter-active, .list-leave-active {
  transition: all 0.3s ease;
}
.list-enter-from, .list-leave-to {
  opacity: 0;
  transform: translateX(30px);
}
.list-leave-active {
  position: absolute;  /* Enable FLIP for remaining items */
}
.list-move {
  transition: transform 0.3s ease;  /* Animate repositioning */
}
</style>
```

The `tag` prop specifies the wrapper element (defaults to no wrapper in Vue 3).

## Appear Transition

Apply transition on initial render:

```vue
<Transition name="fade" appear>
  <p>I animate on page load</p>
</Transition>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Transition wrapping multiple root elements | `<Transition>` only works with a single child |
| Missing `key` on dynamic components | Add `:key` to force re-render and trigger transition |
| List items jumping during leave | Add `position: absolute` to `.list-leave-active` |
| JS hooks not completing | Always call `done()` in `@enter` and `@leave` |
