# Performance Optimization

Strategies for keeping Vue/Nuxt applications fast — from reactivity tuning to bundle optimization.

## Reactivity Optimization

### shallowRef / shallowReactive

Avoid deep reactivity for large objects that change as a whole:

```ts
// Deep reactivity — tracks every nested property
const state = ref({ users: [{ name: 'Alice', settings: { theme: 'dark' } }] })

// Shallow — only tracks .value replacement, not nested changes
const state = shallowRef({ users: [...] })

// To trigger update, replace the entire value:
state.value = { ...state.value, users: newUsers }
```

Use when:
- Large arrays/objects from API responses
- Data that's replaced wholesale, not mutated
- Third-party data structures (Map, Set, class instances)

### triggerRef

Force update a shallowRef after manual mutation:

```ts
const data = shallowRef({ count: 0 })
data.value.count++    // Won't trigger update
triggerRef(data)       // Now it will
```

## Template Optimization

### v-once

Render once, never re-render:

```vue
<template>
  <footer v-once>
    <p>Copyright 2026 Vue Advanced Concepts</p>
  </footer>
</template>
```

### v-memo

Memoize template sub-trees:

```vue
<template>
  <div v-for="item in longList" :key="item.id" v-memo="[item.name, item.active]">
    <ExpensiveChild :item="item" />
  </div>
</template>
```

Only re-renders when `item.name` or `item.active` changes.

## Component Lazy Loading

### Lazy Components

```vue
<script setup>
const HeavyEditor = defineAsyncComponent(() => import('./HeavyEditor.vue'))
</script>

<template>
  <HeavyEditor v-if="showEditor" />
</template>
```

### Nuxt Lazy Prefix

```vue
<template>
  <!-- Automatically code-split and lazy-loaded -->
  <LazyHeavyChart v-if="showChart" />
</template>
```

Any component prefixed with `Lazy` in Nuxt is automatically lazy-loaded.

## Code Splitting

### Route-Level Splitting

Nuxt does this automatically. In Vue Router:

```ts
const routes = [
  { path: '/dashboard', component: () => import('./Dashboard.vue') },
]
```

### Manual Dynamic Imports

```vue
<script setup>
let markdown: any = null

async function loadMarkdownParser() {
  const { marked } = await import('marked')  // Only loads when called
  markdown = marked
}
</script>
```

## Virtual Scrolling

For long lists (1000+ items), only render visible items:

```vue
<script setup>
import { useVirtualList } from '@vueuse/core'

const allItems = ref(Array.from({ length: 10000 }, (_, i) => ({ id: i, name: `Item ${i}` })))
const { list, containerProps, wrapperProps } = useVirtualList(allItems, { itemHeight: 40 })
</script>

<template>
  <div v-bind="containerProps" style="height: 400px; overflow: auto;">
    <div v-bind="wrapperProps">
      <div v-for="{ data } in list" :key="data.id" style="height: 40px;">
        {{ data.name }}
      </div>
    </div>
  </div>
</template>
```

## Bundle Analysis

### Analyze Bundle Size

```bash
npx nuxi analyze
```

This opens an interactive treemap showing what's in your bundle.

### Common Bundle Bloat Sources

| Source | Solution |
|---|---|
| Moment.js (~300KB) | Use dayjs (~2KB) or date-fns |
| Lodash (full) | Import specific functions: `import debounce from 'lodash/debounce'` |
| Large icon sets | Only import icons you use |
| Unused CSS | Tailwind purges automatically; check other frameworks |

## Nuxt Performance Features

### Payload Extraction

```ts
export default defineNuxtConfig({
  experimental: {
    payloadExtraction: true  // Extracts page payloads for caching
  }
})
```

### Route Rules for Caching

```ts
export default defineNuxtConfig({
  routeRules: {
    '/': { prerender: true },
    '/api/**': { cache: { maxAge: 3600 } },           // Cache API for 1 hour
    '/blog/**': { isr: 3600 },                         // Incremental Static Regeneration
    '/static/**': { headers: { 'cache-control': 'max-age=31536000' } },
  }
})
```

## Lighthouse Checklist

| Metric | Target | Key Optimization |
|---|---|---|
| FCP (First Contentful Paint) | < 1.8s | SSR/SSG, font preloading |
| LCP (Largest Contentful Paint) | < 2.5s | Optimize hero image, preload critical assets |
| CLS (Cumulative Layout Shift) | < 0.1 | Set image dimensions, avoid dynamic content above fold |
| TBT (Total Blocking Time) | < 200ms | Code split, lazy load, reduce JS |

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Premature optimization | Profile first, optimize bottlenecks |
| Over-using shallowRef | Only needed for large, frequently replaced data |
| v-memo with wrong dependencies | Must include all reactive values used in block |
| Not lazy-loading below-fold content | Use `LazyComponent` or `v-if` with IntersectionObserver |
