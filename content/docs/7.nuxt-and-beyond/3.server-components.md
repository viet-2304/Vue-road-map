# Server Components

Nuxt Server Components (Islands Architecture) render components entirely on the server, sending only HTML to the client — zero JavaScript for those components.

## What Are Server Components?

Regular components are hydrated on the client (JavaScript executes to make them interactive). Server components skip hydration entirely:

| Feature | Regular Component | Server Component |
|---|---|---|
| Renders on server | Yes (SSR) | Yes |
| Hydrates on client | Yes | No |
| Client-side interactivity | Yes | No |
| JavaScript sent to client | Yes | No |
| Best for | Interactive UI | Static content, heavy renders |

## Creating Server Components

Add the `.server.vue` suffix:

```vue
<!-- components/HeavyChart.server.vue -->
<script setup>
// This code runs only on the server
const data = await $fetch('/api/analytics')
const chartHtml = renderChart(data)  // Heavy computation on server
</script>

<template>
  <div class="chart" v-html="chartHtml"></div>
</template>
```

Use it like any component — Nuxt handles the rest:

```vue
<template>
  <div>
    <h1>Dashboard</h1>
    <HeavyChart />  <!-- Rendered on server, no JS shipped -->
    <InteractiveForm />  <!-- Normal component, fully interactive -->
  </div>
</template>
```

## NuxtIsland Component

Under the hood, server components use `<NuxtIsland>`. You can use it directly:

```vue
<template>
  <NuxtIsland name="HeavyChart" :props="{ period: 'monthly' }" />
</template>
```

## Passing Props

Server components accept props like regular components:

```vue
<!-- components/UserCard.server.vue -->
<script setup>
const props = defineProps<{
  userId: number
}>()

const user = await $fetch(`/api/users/${props.userId}`)
</script>

<template>
  <div class="card">
    <h2>{{ user.name }}</h2>
    <p>{{ user.email }}</p>
  </div>
</template>
```

```vue
<template>
  <UserCard :user-id="42" />
</template>
```

## Enabling Islands

Enable in `nuxt.config.ts`:

```ts
export default defineNuxtConfig({
  experimental: {
    componentIslands: true
  }
})
```

## Use Cases

### 1. Heavy Content Rendering

```vue
<!-- components/MarkdownRenderer.server.vue -->
<script setup>
import { marked } from 'marked'

const props = defineProps<{ content: string }>()
const html = marked(props.content)  // Heavy parsing on server only
</script>

<template>
  <div class="prose" v-html="html"></div>
</template>
```

### 2. Personalized Content

```vue
<!-- components/RecommendedProducts.server.vue -->
<script setup>
const props = defineProps<{ userId: number }>()
const products = await $fetch(`/api/recommendations/${props.userId}`)
</script>

<template>
  <div class="grid grid-cols-3 gap-4">
    <div v-for="product in products" :key="product.id">
      <img :src="product.image" :alt="product.name">
      <p>{{ product.name }}</p>
    </div>
  </div>
</template>
```

### 3. Third-Party Widgets

Keep heavy third-party rendering on the server to reduce client bundle.

## Slots in Server Components

Server components support slots for mixing server and client content:

```vue
<!-- components/ServerLayout.server.vue -->
<template>
  <div class="server-layout">
    <header>Server-rendered header</header>
    <main>
      <slot />  <!-- Client content goes here -->
    </main>
  </div>
</template>
```

## Limitations

- **No client-side interactivity** — no `@click`, no reactive updates on client
- **No lifecycle hooks that run on client** — `onMounted` won't fire
- **Props must be serializable** — no functions, no complex objects
- **Slots are limited** — basic slot support, some edge cases may not work

## Client Components Inside Server Components

If you need interactive parts within a server component, use the `.client.vue` suffix for those parts:

```vue
<!-- components/AddToCartButton.client.vue -->
<script setup>
const props = defineProps<{ productId: number }>()

function addToCart() {
  // Client-side interactivity
  cart.add(props.productId)
}
</script>

<template>
  <button @click="addToCart">Add to Cart</button>
</template>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Expecting interactivity from server components | Use `.client.vue` for interactive parts |
| Passing non-serializable props | Only pass strings, numbers, arrays, plain objects |
| Using browser APIs in server components | They run server-side only — no `window`, `document` |
| Performance not improving | Server components help most with heavy rendering or large dependencies |
