# SEO & Meta

Nuxt provides powerful composables for managing meta tags, Open Graph data, and structured data for search engines.

## useHead

Set any `<head>` element reactively:

```vue
<script setup>
useHead({
  title: 'My Page Title',
  meta: [
    { name: 'description', content: 'Page description for search engines' },
    { name: 'robots', content: 'index, follow' },
  ],
  link: [
    { rel: 'canonical', href: 'https://example.com/page' },
  ],
  script: [
    { src: 'https://analytics.example.com/script.js', defer: true },
  ],
})
</script>
```

### Dynamic Titles

```vue
<script setup>
const { data: post } = await useFetch(`/api/posts/${route.params.id}`)

useHead({
  title: () => post.value?.title ?? 'Loading...',
})
</script>
```

### Title Template

Set a global title template in `nuxt.config.ts`:

```ts
export default defineNuxtConfig({
  app: {
    head: {
      titleTemplate: '%s | Vue Advanced Concepts',
    }
  }
})
```

Pages set just the page part: `useHead({ title: 'Home' })` renders "Home | Vue Advanced Concepts".

## useSeoMeta

Type-safe helper for SEO meta tags:

```vue
<script setup>
useSeoMeta({
  title: 'Vue Advanced Concepts',
  description: 'Learn Vue.js with interactive examples',
  ogTitle: 'Vue Advanced Concepts',
  ogDescription: 'Hands-on learning platform',
  ogImage: 'https://example.com/og-image.png',
  ogUrl: 'https://example.com',
  twitterCard: 'summary_large_image',
  twitterTitle: 'Vue Advanced Concepts',
  twitterDescription: 'Learn Vue.js with interactive examples',
  twitterImage: 'https://example.com/og-image.png',
})
</script>
```

### Dynamic SEO Meta

```vue
<script setup>
const { data: article } = await useFetch(`/api/articles/${route.params.slug}`)

useSeoMeta({
  title: () => article.value?.title,
  description: () => article.value?.excerpt,
  ogImage: () => article.value?.coverImage,
})
</script>
```

## Open Graph Tags

Essential OG tags for social media sharing:

```vue
<script setup>
useHead({
  meta: [
    { property: 'og:type', content: 'website' },
    { property: 'og:title', content: 'My App' },
    { property: 'og:description', content: 'App description' },
    { property: 'og:image', content: 'https://example.com/image.png' },
    { property: 'og:url', content: 'https://example.com' },
    { property: 'og:site_name', content: 'My App' },
    { property: 'og:locale', content: 'en_US' },
  ]
})
</script>
```

## Structured Data (JSON-LD)

Help search engines understand your content:

```vue
<script setup>
useHead({
  script: [
    {
      type: 'application/ld+json',
      innerHTML: JSON.stringify({
        '@context': 'https://schema.org',
        '@type': 'Article',
        headline: 'Vue Advanced Concepts',
        author: { '@type': 'Person', name: 'Author Name' },
        datePublished: '2026-03-27',
        description: 'Learn Vue.js with interactive examples',
      }),
    }
  ]
})
</script>
```

### Breadcrumb Structured Data

```ts
useHead({
  script: [{
    type: 'application/ld+json',
    innerHTML: JSON.stringify({
      '@context': 'https://schema.org',
      '@type': 'BreadcrumbList',
      itemListElement: [
        { '@type': 'ListItem', position: 1, name: 'Home', item: 'https://example.com' },
        { '@type': 'ListItem', position: 2, name: 'Docs', item: 'https://example.com/docs' },
        { '@type': 'ListItem', position: 3, name: 'Composables' },
      ]
    })
  }]
})
```

## Robots.txt

Create `public/robots.txt`:

```
User-agent: *
Allow: /
Sitemap: https://example.com/sitemap.xml
```

Or use the `nuxt-simple-robots` module for dynamic control.

## Sitemap

Use `@nuxtjs/sitemap` module:

```ts
export default defineNuxtConfig({
  modules: ['@nuxtjs/sitemap'],
  site: {
    url: 'https://example.com',
  },
})
```

## Canonical URLs

Prevent duplicate content issues:

```vue
<script setup>
const route = useRoute()
const canonicalUrl = `https://example.com${route.path}`

useHead({
  link: [{ rel: 'canonical', href: canonicalUrl }]
})
</script>
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Meta tags not appearing in view-source | Ensure SSR is enabled — client-only meta won't help SEO |
| OG image not showing on social media | Use absolute URLs, minimum 1200x630px |
| Duplicate meta tags | Use `key` property or `useServerSeoMeta` |
| Title template overriding page titles | Use `titleTemplate: null` on specific pages |
