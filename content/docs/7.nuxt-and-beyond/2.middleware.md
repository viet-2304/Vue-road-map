# Nuxt Middleware

Middleware runs code before navigating to a route — authentication guards, redirects, logging, and more.

## Route Middleware

### Named Middleware

Create in `middleware/` directory:

```ts
// middleware/auth.ts
export default defineNuxtRouteMiddleware((to, from) => {
  const auth = useAuthStore()

  if (!auth.isLoggedIn) {
    return navigateTo('/login')
  }
})
```

Use in pages:

```vue
<script setup>
definePageMeta({
  middleware: 'auth'
})
</script>
```

### Inline Middleware

Define directly in a page:

```vue
<script setup>
definePageMeta({
  middleware: [
    function (to, from) {
      const auth = useAuthStore()
      if (!auth.isAdmin) return navigateTo('/')
    }
  ]
})
</script>
```

### Global Middleware

Add `.global` suffix to run on every route:

```ts
// middleware/analytics.global.ts
export default defineNuxtRouteMiddleware((to, from) => {
  if (import.meta.client) {
    trackPageView(to.path)
  }
})
```

## Middleware Execution Order

1. Global middleware (alphabetical by filename)
2. Page-defined middleware (in array order)

Use number prefixes for ordering:

```
middleware/
  01.auth.global.ts     ← Runs first
  02.analytics.global.ts ← Runs second
  admin.ts              ← Named, not global
```

## Return Values

```ts
export default defineNuxtRouteMiddleware((to, from) => {
  // Allow navigation (continue)
  return  // or return undefined

  // Redirect
  return navigateTo('/login')

  // Redirect with options
  return navigateTo('/login', { redirectCode: 301 })

  // External redirect
  return navigateTo('https://example.com', { external: true })

  // Block navigation (abort)
  return abortNavigation()

  // Block with error
  return abortNavigation(createError({ statusCode: 403, message: 'Forbidden' }))
})
```

## Auth Guard Pattern

```ts
// middleware/auth.ts
export default defineNuxtRouteMiddleware((to) => {
  const auth = useAuthStore()

  // Public pages
  const publicPages = ['/login', '/register', '/']
  if (publicPages.includes(to.path)) return

  // Check auth
  if (!auth.token) {
    return navigateTo({
      path: '/login',
      query: { redirect: to.fullPath }
    })
  }

  // Check role
  if (to.meta.requiredRole && auth.user?.role !== to.meta.requiredRole) {
    return abortNavigation(createError({ statusCode: 403 }))
  }
})
```

## Server Middleware

Runs on the server only, for API routes and server-side logic:

```ts
// server/middleware/log.ts
export default defineEventHandler((event) => {
  console.log(`[${event.method}] ${getRequestURL(event)}`)
})
```

Server middleware runs for ALL server requests (pages and API routes).

### Server API Routes

```ts
// server/api/users.get.ts
export default defineEventHandler(async (event) => {
  const users = await db.query('SELECT * FROM users')
  return users
})

// server/api/users.post.ts
export default defineEventHandler(async (event) => {
  const body = await readBody(event)
  const user = await db.insert('users', body)
  return user
})
```

## Common Pitfalls

| Pitfall | Solution |
|---|---|
| Middleware causing infinite redirect | Exclude the redirect target from middleware checks |
| Using browser APIs in middleware | Check `import.meta.client` — middleware runs on server too |
| Middleware not running on initial load | Global middleware runs on both client and server navigation |
| Heavy async operations blocking navigation | Show a loading state, keep middleware fast |
